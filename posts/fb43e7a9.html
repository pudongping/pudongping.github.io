var、let、const 的区别

  • var 声明的变量是全局作用域（var 声明的全局作用域变量属于 window 对象，可以通过 window.variable 的方式去访问）或函数作用域，而 let 和 const 是块作用域。
  • var 声明的变量可以在其范围内更新和重新声明，而 let 变量可以被更新但不能重新声明，const 常量既不能更新也不能重新声明。
  • var 、 let 、const 都会被提升到作用域的顶端（变量提升），但是，虽然会使用 undefined 初始化 var 定义的变量，但是不会初始化 let 和 const 声明的变量。
  • 尽管可以在不初始化的情况下声明 var 和 let 变量，但是在声明期间必须初始化 const 。

变量赋值只是变量名和变量值的映射关系，变量名也只是会保存变量值的内存地址，const 修饰之后，不允许修改已经保存的变量值的内存地址，但是允许改变变量值内部的属性。


const boy = {
    sex: 1,
    age: 26
};

// 这样会报错
const boy = {
    sex: 0,
    age: 18
};

// 但是可以这么操作
boy.sex = 0;
boy.age = 18;

var 可以声明为全局或者函数内部的变量作用域，但是没有块级别作用域，如 for 、 while 循环时，使用 let 声明变量时，for 、while 循环就有作用域了。


'use strict';

function foo() {
    for (var i=0; i<100; i++) {
        //
    }
    i += 100; // 仍然可以引用变量i
}

let 可以申明一个块级作用域的变量


'use strict';

function foo() {
    var sum = 0;
    for (let i=0; i<100; i++) {
        sum += i;
    }
    // SyntaxError:
    i += 1;
}

MVVM （model–view–viewmodel）

  1. View 层，（视图层，在我们前端开发中，通常就是 DOM 层）主要的作用是显示信息
  2. Model 层，（数据层，来源于接口或者写死的数据）
  3. ViewModel 层，（视图模型层）视图模型层是 View 和 Model 沟通的桥梁。一方面它实现了 Data Binding（数据绑定），将 Model 的改变实时反应到 View 中，另一方面它实现了 DOM Listener（DOM 监听），当 DOM 发生一些事件（点击、滚动、touch 等）时，可以监听到，并在需要的情况下改变对应的 Data。

数组中哪些方法是响应式的？


<!DOCTYPE html>
<html lang="zh-CN">
	<head>
		<meta charset="utf-8">
		<title></title>
	</head>
	<body>
		
		<div id="app">
			
			<ul>
				<ul v-for="item in letters">{{item}}</ul>
			</ul>

			<button @click="btnClick()">去改变值</button>
		</div>

		
	</body>
</html>

<script type="text/javascript" src="vue-js/vue.js"></script>

<script type="text/javascript">
		
	const app = new Vue({
		el: '#app',
		data: {
			letters: ['a', 'b', 'c', 'd']
		},
		methods: {
			btnClick() {

				// 1.push() 方法 ：尾部追加值
				// this.letters.push('aaa')
				// this.letters.push('aaa', 'bbb', 'ccc')

				// 2. pop() 方法，删除数组中的最后一个元素
				// this.letters.pop()

				// 3. shift() 方法，删除数组中的第一个元素
				// this.letters.shift()

				// 4. unshift() 方法，在数组最前面添加元素
				// this.letters.unshift('aaa')
				// this.letters.unshift('aaa', 'bbb', 'ccc')

				// 5. splice() 方法，可以 删除元素/插入元素/替换元素
				// 删除元素：第二个参数传入你要删除几个元素（如果没有传，就删除后面所有的元素）
				// 替换元素：第二个参数，表示我们要替换成几个元素，后面是用于替换前面的元素
				// 插入元素：第二个参数，传入 0，并且后面跟上要插入的元素
				// this.letters.splice(1)
				// this.letters.splice(1, 2, 'm', 'n', 'l', 'x')
				// this.letters.splice(1, 0, 'x', 'y', 'z')

				// sort() 排序
				// reverse() 数组反转

				// 以上方法，都是响应式的，唯独下面以索引值对数组进行修改，不是响应式的
				this.letters[1] = 'hhhh';

			}
		},

	})
</script>

～ 波浪线的使用

波浪线后面跟数字时，返回值为该数字取反（正数变负数，负数变正数）然后减去 1


~1  // -2
~2  // -3
~-1  // 0
~-2  // 1
~0  // -1

结论：只有 ~-1 时结果才会为 0 ，其他任何数值结果都不为 0，（波浪线）~ 与 indexOf() 方法连用，如果父字符串不包含子字符时返回 0 或者 false，多用于条件判断。
